# Design Decisions

Accepted decisions that still matter to contributors live here.

Use this directory when you need durable rationale for a behavior that looks
arbitrary in the code or reference docs.

## Current Decisions

- [Idempotency-Key Migration](idempotency-key-migration.md) explains the v2 to
  v3 carry-forward strategy for import deduplication data.

## When To Read This Folder

- when a migration or storage behavior seems oddly specific
- when a reference doc points to a historical rationale
- when you need to know whether a design choice was deliberate or accidental
